### 1. Fork the Repository
```bash

cd python-finance-academy
```

### 2. Create a Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Follow our code style guidelines
- Add tests if applicable
- Update documentation

### 4. Test Your Changes
```bash

pip install -r requirements.txt

# Run notebooks to verify they work
jupyter nbconvert --execute curriculum/**/*.ipynb
```

### 5. Commit Your Changes
```bash
git add .
git commit -m "Add: description of your changes"
```

### 6. Push and Create PR
```bash
git push origin feature/your-feature-name
```
Then open a Pull Request on GitHub.

## ğŸ“ Style Guidelines

### Code Style
- Follow PEP 8 for Python code
- Use meaningful variable names
- Add comments for complex logic
- Keep functions small and focused

### Notebook Style
- Start with a clear title and learning objectives
- Use markdown cells to explain concepts
- Include practical examples
- Add exercises at the end
- Provide solutions in separate cells

### Commit Messages
Use conventional commits:
- `Add:` for new features
- `Fix:` for bug fixes
- `Update:` for updates to existing content
- `Docs:` for documentation changes
